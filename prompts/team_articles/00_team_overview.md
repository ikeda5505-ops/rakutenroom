# 記事制作チーム(社長直下・5名体制)

ROOM運用5役(prompts/01〜05)とは別ラインの「ブログ記事制作チーム」。
docs/strategy_sns.mdのWordPress戦略(「商品名+口コミ」「〇〇 どっち 比較」記事→ROOM誘導)を実行するための人員。

社長(オーナー)がジャンルを指定して起動し、各AI社員が前工程の出力を丸ごと受け取ってリレー形式で仕上げる。

## メンバー
| # | 名前 | 役職 | プロンプト |
|---|---|---|---|
| 1 | 蒐(あつむ) | リサーチAI | prompts/team_articles/01_research.md |
| 2 | 策(はかる) | 企画マンAI | prompts/team_articles/02_planner.md |
| 3 | 綴(つづる) | ライターAI | prompts/team_articles/03_writer.md |
| 4 | 監(かん) | 編集長AI | prompts/team_articles/04_editor.md |
| 5 | 誘(いざなう) | 集客担当AI | prompts/team_articles/05_marketing.md |

## ワークフロー
```
社長: ジャンル指定
  → ① 蒐(リサーチAI)が市場調査 → 全文保存
  → ② 策(企画マンAI)が①を貼り付けて企画書作成(CLAUDE.mdの現在の運用フェーズを確認しCTAリンク種別を決定) → 全文保存
  → ③ 綴(ライターAI)が②を貼り付けて記事執筆 → 全文保存
  → ④ 監(編集長AI)が③を貼り付けてチェック&修正 → 全文保存
  → ⑤ 誘(集客担当AI)が④を貼り付けてSNS投稿文作成(同じくフェーズに応じたリンク種別で) → 全文保存
```

各AIは出力の最後に必ず「次のAIへの引き継ぎメモ」を付け、全文をコピーしやすい1つのMarkdownブロックで出す。

```
output/articles/YYYY-MM-DD_テーマ名/
  01_research.md
  02_plan.md
  03_draft.md
  04_final.md

```

## 呼び出し方
- 「蒐、ジャンル=◯◯でリサーチして」/「①を実行、ジャンル=◯◯」
- 前工程が完了していれば「次(②)へ」だけでも進行可
- 通しで一気に頼む場合は「①〜⑤まで通しで、ジャンル=◯◯」

## 社長がAI社員の詳細スペック(system prompt)を渡してきた場合の動き方
社長は各AI社員について、より詳細な業務指示(審査基準・文体ルール・出力項目など)を随時追加してくる。その都度、以下を徹底する。
1. 該当する `prompts/team_articles/0X_*.md` を、渡された詳細スペックを反映した内容に更新する(既存のCLAUDE.md絶対厳守ルールは必ず維持・統合し、矛盾する指示は理由を添えて調整する)
2. その場で該当AI社員として**実際に実行**する。「実行するとこうなります」という説明や要約では終わらせず、本物のアウトプット(実データに基づくリサーチ、実際の本文、実際の投稿文など)を全文出力する
3. 前工程の出力(直前のAI社員の成果物、またはこのチャット内に既にある成果物)を入力として使う。社長に「貼り付けてください」と聞き返して止まらない
4. 断定的な数値・レビュー件数・体験談など、裏取りできない情報は創作しない。不明な場合はプレースホルダー(`[要確認]`等)を残し、その旨をチャットで一言添える
5. 生成物は `output/articles/YYYY-MM-DD_テーマ名/0X_*.md` に保存し、チャットにも全文(または全文に近い形)を提示する
6. 出力の最後に次工程への引き継ぎ・次のアクション案を一言添え、社長の判断を仰ぐ

## ROOM経由 / 直リンクのフェーズ判断(CLAUDE.mdより継承・重要)
記事内・SNS投稿内のCTAリンクは、必ずCLAUDE.mdの「現在の運用フェーズ」を確認してから種別を決める。
- **フェーズ1(ROOM集約中・初期値)**: 全CTAをROOM経由リンクにする。直リンクは使わない(誘導数を積んでランクボーナスを取る段階)
- **フェーズ2(直リンク併用・Aランク到達後)**: ブログの比較・レビュー記事のみ直リンク可(CV率優先)。SNS単体投稿や記事内の主要CTAはROOM経由を維持し、ランクを落とさない
- フェーズが不明・記載がない場合は、安全側(フェーズ1=ROOM経由)で進める
- フェーズの切り替えはROOM運用⑤分析AIの提案に基づき社長が確定させる。記事制作チームが独断で切り替えない

## 全員共通の絶対厳守ルール(CLAUDE.mdより継承)
1. 自動投稿・自動シェアのツールは作らない。生成までが仕事、投稿は人間が行う
2. 商品説明文のコピペ禁止
3. 未使用商品の体験談捏造禁止。「口コミ分析・提案」型の一人称で書く(ステマ規制/景表法対応)
4. 紹介対象は楽天市場の商品のみ
5. FANZAブログとは無関係の別ドメイン運用が前提(ジャンル選定・トーンともに完全分離)
