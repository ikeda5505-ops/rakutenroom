# 楽天ROOM収益化プロジェクト

## リポジトリの技術的な性質
- ビルド/lint/テストの類は存在しない。このリポジトリは「Markdownプロンプトテンプレート」+「そこから生成されたコンテンツ」を管理するだけの純粋なコンテンツ運用リポジトリ
- 唯一のコードは `assets/make_brand.py`(Python + Pillow でブランドアイコン/ヘッダー画像を生成)。依存関係を宣言する requirements.txt 等はない。ローカルで動かす場合は `pip install pillow` に加え、スクリプトがハードコードしている `/usr/share/fonts/opentype/noto/NotoSerifCJK-Bold.ttc` 等のNoto CJKフォントパスが必要。出力先も `/home/claude/room_icon.png` 等にハードコードされているため、リポジトリ内で再生成する際は保存先パスを `assets/room_icon.png` / `assets/room_header.png` に書き換えてから実行すること
- 「実行」とは常に「このリポジトリのMarkdownプロンプト(prompts/配下)をチャット上でAIに演じさせ、その場でテキスト成果物を生成すること」を指す。ここにAPI呼び出しや自動投稿を行うプログラムは存在しないし、作ってはいけない(下記コンプライアンスルール1)

## プロジェクト概要
- オーナー: Masatsugu
- 目的: 楽天ROOMアフィリエイトの半自動運用による収益化(月1万円→拡大)
- 位置づけ: アダルト事業(FANZA)とは完全分離の健全ジャンル収益源
- ROOM屋号: くらし研究室|暮らしを快適にするモノ選び
- コンセプト: 季節の悩みを先回りで解決する、暮らし快適化ROOM
- ブランドカラー: クリーム #F7F3EC / セージ #8FA98F / テラコッタ #C97B5A / インク #3E4A3D

## 絶対厳守ルール(コンプライアンス)
1. 楽天ROOMへの自動投稿・自動フォロー・自動いいねのコード/ツールは作らない(規約違反=垢BAN)。自動化するのは「生成」まで。投稿は必ず人間が行う
2. 商品説明文のコピペ投稿文は作らない(ランク審査落ち)
3. 未使用商品の「使ってみた」捏造体験談は書かない(ステマ規制/景表法)。「口コミ分析・提案」型の一人称で書く
4. 紹介対象は楽天市場の商品のみ

## AIチーム構成(5役・ROOM運用ライン)
各役割のプロンプトは prompts/ 配下。呼び出し方:「①を実行」等
- ① リサーチ (prompts/01_research.md) - 週次の商品10選
- ② コピーライティング (prompts/02_copywriting.md) - 投稿文3パターン生成
- ③ ROOM設計 (prompts/03_room_design.md) - プロフィール/コレクション
- ④ 運用カレンダー (prompts/04_calendar.md) - 週間投稿スケジュール
- ⑤ 分析 (prompts/05_analytics.md) - 週次レポート分析、ROOM経由/直リンクのCV比較、フェーズ切替提案、翌週最適化

## 記事制作チーム構成(5名・社長直下・ブログ制作ライン)
docs/strategy_sns.mdのWordPress戦略を実行する別ライン。プロンプトは prompts/team_articles/ 配下。
前工程の出力全文を次工程に貼り付けるリレー形式。詳細は prompts/team_articles/00_team_overview.md 参照。
- 蒐(あつむ) リサーチAI (01_research.md) - ジャンル指定で市場調査
- 策(はかる) 企画マンAI (02_planner.md) - リサーチを貼り付けて企画書作成(ROOM誘導/直リンクの判断を含む)
- 綴(つづる) ライターAI (03_writer.md) - 企画書を貼り付けて記事執筆
- 監(かん) 編集長AI (04_editor.md) - 原稿を貼り付けてチェック&修正
- 誘(いざなう) 集客担当AI (05_marketing.md) - 完成原稿を貼り付けてSNS投稿文作成
- 保存先: output/articles/YYYY-MM-DD_テーマ名/ に01〜05を全文保存

## 現在の運用フェーズ(⑤分析AIが週次で更新)
- **現フェーズ: フェーズ1 = ROOM集約中**(初期値。Aランク未到達のため)
- 記事制作チームの②企画・⑤SNS投稿でCTAリンクを選ぶ際は、このフェーズを必ず確認する
  - フェーズ1(ROOM集約中): ブログ・SNSのCTAは全てROOM経由リンクにする(直リンク不可)
  - フェーズ2(直リンク併用): Aランク到達後。ブログの比較・レビュー記事のみ直リンク可、それ以外(SNS単体投稿等)はROOM経由を維持
- フェーズが切り替わったら、⑤分析AIの提案を受けてこのセクションを更新すること

## 運用サイクル
- 月曜: ①で商品10選 → ②で投稿文一括生成 → content/ に週別mdで保存
- 毎日: 生成済み投稿文を朝7時/昼12時/夜21時に手動投稿(15分)
- 土曜: 楽天アフィリエイトレポートを data/ にCSVで置く → ⑤で分析 → ④で翌週カレンダー更新

## KPI
- 初月: Bランク確定(#オリジナル写真の月間指定枚数達成) + クリック数のみ追う
- 3ヶ月目標: 月1,500〜2,000クリック(Aランク圏)
- ランクボーナス: B+1% / A+2% / S+3%(1商品上限1,000円)

## ディレクトリとファイル命名規則(実例)
- prompts/                : ROOM運用5役のプロンプトテンプレート(01_research.md〜05_analytics.md)
- prompts/team_articles/  : 記事制作チーム5名のプロンプトテンプレート(00_team_overview.md + 01_research.md〜05_marketing.md)
- docs/    : 初心者マニュアル(getting_started.md=完全手順、manual.md=要点版)、profile.md(プロフィール文確定版)、strategy_sns.md(外部SNS戦略)
- content/ : ②が生成した投稿文ストック(週別)。ファイル名 `posts_YYYY-MM-DD.md`(例: content/posts_2026-07-14.md)。週の起点日でファイルを分け、1商品につきA/B(/C)複数パターンをまとめて1ファイルに保存
- assets/  : ブランド画像(room_icon.png/room_header.png)、design_philosophy.md(ビジュアルコンセプト文)、make_brand.py(生成スクリプト)
- data/    : 楽天アフィリエイトレポートCSV置き場(README.mdのみ、CSV自体は都度手動配置)
- output/  : ①④⑤の成果物。ファイル名は `weekly_research_YYYY-MM-DD.md`(①の商品10選)、`calendar_YYYY-MM-DD_week.md`(④の週間カレンダー)など、担当AIと対象週が分かる名前にする
- output/articles/ : 記事制作チームの成果物。1記事1フォルダ `output/articles/YYYY-MM-DD_テーマ名/` に `01_research.md` `02_plan.md` `03_draft.md` `04_final.md` `05_sns.md` を保存(ファイル名は工程を表す名詞に変わる点に注意。プロンプトファイル名の `01_research.md`〜`05_marketing.md` とは対応するが同名ではない)

- 日本語で応答
- 新しい施策を提案する際は必ずコンプライアンス(上記ルール1-4)を先に確認
- **AI社員(prompts/team_articles/配下)の詳細スペックを社長から渡された場合**: ①該当プロンプトファイルを更新 ②その場で本物のアウトプットとして即実行(説明や要約で終わらせない) ③前工程の出力は聞き返さずこのチャット内の直近成果物を使う ④裏取りできない数値・体験談は創作せずプレースホルダー化 ⑤output/配下に保存しチャットにも全文提示 ⑥次工程への引き継ぎを一言添える。詳細は prompts/team_articles/00_team_overview.md 参照

## 楽天アフィリエイトとROOMの関係(重要な前提)
- 楽天アフィリエイト=報酬制度の本体(リンク発行/成果計測/レポート/支払い)
- 楽天ROOM=その上に乗るSNS。「コレ!」投稿は自動でアフィリエイトリンク化
- 報酬・ID・レポートは一つ。露出手段が「ROOM経由」と「直リンク」の2系統

## 導線の使い分けルール
1. Aランク到達まで: 全導線(X/ブログ)をROOMへ集約。クリックを誘導数に積みランクボーナスを取る
2. Aランク安定後: ブログの比較・レビュー記事のみ直リンクへ切替(CV率優先)。ROOMランクは残り導線で維持
3. レポートは楽天アフィリエイト管理画面で一元確認。data/のCSVでROOM経由/直リンクを比較分析(⑤の仕事)
